this can do phone and PC are not in the same LAN,
they're connected by usb cable, I wonder if this can bring a good stuff to ssh -XY, or tigerVNC?

on termux
proot-distro login debian-buster
useradd -m john
passwd john
apt install openssh-server
vim /etc/ssh/sshd_config
Port 60022
#enable root login or not
/etc/init.d/ssh start

connect your phone to PC by usb cable

on PC
adb shell #check if it's connected, if it's not, then choose "file transform" over "charge only" on use option
adb forward tcp:60022 tcp:60022
ssh -p 60022 john@localhost # don't use root, it can't be used if you enabled it in /etct/ssh/sshd_config

on termux 

apt install xfce4 xfce4-goodies
apt install tightvncserver
vncserver #set your vnc password
vncserver -kill :1
vim  ~/.vnc/xstartup
xrdb $HOME/.Xresources
startxfce4 &

chmod +x ~/.vnc/xstartup
vncserver
